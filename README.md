# DaisyPharmacy

Simple Java Swing pharmacy management application (NetBeans/Ant project).

**This README explains how to set up, build and run the project locally.**

**Prerequisites**
- Java JDK 8 or later installed and `java`/`javac` on PATH.
- Apache Ant (optional, used if you want to build from command line). NetBeans includes Ant implicitly.
- MySQL server running locally (or accessible) — the app expects a database named `pharmacy`.
- (Optional) NetBeans IDE for easiest development and running of the project.

**Important files**
- `src/` — Java sources and NetBeans generated form files.
- `build.xml` — Ant build file (NetBeans generated).
- `src/dao/ConnectionProvider.java` — JDBC connection settings (default: `jdbc:mysql://127.0.0.1:3306/pharmacy`, user `root`, empty password).
- `src/dao/Tables.java` — contains SQL statements (some commented) used to create tables.

**Database setup**
1. Start MySQL server and log in as a user with privileges to create databases/tables.

2. Create the database:

```sql
CREATE DATABASE IF NOT EXISTS pharmacy;
USE pharmacy;
```

3. Create the tables used by the app (these statements are copied from `src/dao/Tables.java`):

```sql
CREATE TABLE IF NOT EXISTS appuser (
  appuser_pk INT AUTO_INCREMENT PRIMARY KEY,
  userRole VARCHAR(200),
  name VARCHAR(200),
  dob VARCHAR(50),
  mobileNumber VARCHAR(50),
  email VARCHAR(200),
  username VARCHAR(200),
  password VARCHAR(100),
  address VARCHAR(200)
);

CREATE TABLE IF NOT EXISTS medicine (
  medicine_pk INT AUTO_INCREMENT PRIMARY KEY,
  uniqueId VARCHAR(200),
  name VARCHAR(200),
  companyName VARCHAR(200),
  quantity BIGINT,
  price BIGINT
);

CREATE TABLE IF NOT EXISTS bill (
  bill_pk INT AUTO_INCREMENT PRIMARY KEY,
  billId VARCHAR(200),
  billDate VARCHAR(50),
  totalPaid BIGINT,
  generatedBy VARCHAR(50)
);
```

4. (Recommended) Insert an initial admin user so you can log in:

```sql
INSERT INTO appuser (userRole, name, dob, mobileNumber, email, username, password, address)
VALUES ('Admin', 'Admin', '1990-01-01', '0000000000', 'admin@example.com', 'admin', 'admin', 'Office');
```

5. If your MySQL username/password or host/port differs from the defaults, update `src/dao/ConnectionProvider.java` accordingly.

**Build & Run**
Option A — Using NetBeans (recommended):
- Open NetBeans and choose `File -> Open Project...` and select the project folder (`DaisyPharmacy`).
- Clean and Build the project, then Run. The NetBeans run configuration runs the `Login` form which is the main UI entry point.

Option B — Using Ant and command line:
- From the project root (where `build.xml` exists), run:

```bash
ant clean
ant
```

- After a successful build, run the `Login` class (the project uses default package for forms). From the project root run:

```bash
java -cp build/classes Login
```

Notes:
- If the build requires additional JARs, put them in a `lib/` directory or configure the classpath. NetBeans usually manages project libraries for you.
- If you get `ClassNotFoundException` for MySQL driver, ensure the MySQL Connector/J JDBC driver jar is on the classpath (e.g., `mysql-connector-java-x.x.x.jar`). With NetBeans, add the JDBC driver to the project's Libraries. For command-line runs, add `-cp lib/mysql-connector-java.jar:build/classes`.

Example command-line run with connector in `lib/`:

```bash
java -cp lib/mysql-connector-java.jar:build/classes Login
```

**Troubleshooting**
- "Cannot connect to DB" — check MySQL is running and `ConnectionProvider` credentials (host, port, database, username, password) are correct.
- "No suitable driver" or `ClassNotFoundException: com.mysql.cj.jdbc.Driver` — add MySQL Connector/J to classpath.
- GUI not appearing — make sure there are no runtime exceptions; run from IDE to see stack traces.
